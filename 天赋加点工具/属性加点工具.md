# 属性加点工具

## 需求
### 核心需求
1. 生成一个属性加点工具，可以进行加点，预览表现。用html生成
2. 属性有两大类，基础属性（5种元素）、一级属性（元素属性 + 其他属性）。
3. 任意基础属性均可自由加点（**仅允许加点，不允许减点**；重置后可重新分配），但每种元素加点上限为 100 点
4. 每次加点基础属性，一级属性自动按配置系数增长
5. 加点时需有预览，采用"xx+y"格式显示，y为所有未确认点数的总增量。参考卧龙做法
6. 加点为批量确认模式：自由调整多种元素后，点击"确认"按钮一次性提交生效
7. 确认后的数据持久化到浏览器缓存（localStorage），下次打开自动恢复。点击"重置"恢复无加点初始状态
8. 一级属性中受基础加点影响的属性纯展示展示，不可手动编辑

> **技术实现**：本工具为独立的 HTML 单页，与项目内 `数值伤害曲线` 工具为两个独立的 HTML 页面，无统一入口，各自可单独打开运行。

### 交互规则

1. **加点**：基础属性（5种元素）通过 `+` 按钮自由加点，修改后立即进入"预览状态"
2. **高亮联动**：鼠标移到左侧基础元素上，对右侧受该元素影响的一级属性进行高亮显示
   - 触发对象：左侧基础属性（火/土/金/水/木）
   - 高亮范围：中间列和右侧列中所有受该元素加点影响的属性
3. **预览格式**：
   - 基础属性显示："当前值" → 点击 `+` 后显示"已加点数+1"
   - 受影响的属性显示：`xx+y` 格式，其中 `xx` 为已确认值，`y` 为所有未确认加点带来的总增量
   - 不受影响的属性保持正常显示
4. **确认机制（批量模式）**：
   - 用户自由调整多种元素后，点击"确认加点"按钮，一次性将所有预览增量合并到最终值
   - 确认后，预览标记清除，`xx+y` 变为合并后的新值
5. **tooltip说明**：鼠标悬停任意属性名上时，显示该属性的详细说明文案（可配置）
6. **公式描述值**：
   - 防御力：tooltip中显示"减伤率 x%"（减伤率 = def / (def + 500) × 100%）
   - 各元素抗性：tooltip中显示"xx元素伤害减免 x%"（伤害减免 = 抗性 × 1%）
7. **实时统计**：显示总的已确认元素加点数量，随确认操作更新
8. **数据持久化**：
   - 每次点击"确认加点"后，将当前所有属性值保存到 localStorage
   - 页面加载时自动从 localStorage 恢复上次确认的状态
   - 提供"重置"按钮，清除所有加点（包括预览状态），恢复为初始配置值
9. **一级属性不可编辑**：所有一级属性（中间列、右侧列）为纯展示展示，不可手动输入

### 数据配置结构（核心）

> **外部配置文件**：以下所有 JSON 配置均为独立的外部 `.json` 文件，与 HTML 同目录放置。工具启动时通过 `fetch` 动态加载，修改配置无需改动代码，刷新页面即生效。
>
> | 文件名 | 用途 |
> |--------|------|
> | `elementConfig.json` | 基础元素及系数配置 |
> | `attributeInitConfig.json` | 一级属性初始值配置 |
> | `attributeMaxConfig.json` | 一级属性上限配置 |
> | `computedAttrConfig.json` | 计算属性公式配置 |

### 显示取整规则

所有数值在界面上展示时，必须按以下规则进行格式化取整。计算属性（元素强度）的取整规则继承其依赖属性的取整方式。

#### 默认规则
- **通用数值**：保留两位小数（如 `12.50`、`3.00`）
- **百分比类属性**：保留两位小数的百分比，后缀 `%`（如 `5.00%`、`150.00%`）

#### 特殊规则（覆盖默认）

| 属性 key | 属性名 | 取整方式 | 说明 |
|---------|-------|---------|------|
| `qiCap` | 真气上限 | `floor`（向下取整） | 对原始值执行 `Math.floor()`，不保留小数位 |
| `critRate` | 暴击率 | `percent2` | 保留两位小数的百分比（如 `5.00%`、`5.50%`） |
| `critDamage` | 暴击伤害 | `percent2` | 保留两位小数的百分比（如 `150.00%`） |

#### 配置字段说明

每个一级属性可在 `attributeInitConfig` 中通过 `roundType` 字段声明取整方式：

| roundType 值 | 含义 | 格式化逻辑 |
|-------------|------|-----------|
| 不填（默认） | 默认两位小数 | `value.toFixed(2)` |
| `"decimal2"` | 保留两位小数 | `value.toFixed(2)` |
| `"floor"` | 向下取整 | `Math.floor(value)` |
| `"percent2"` | 保留两位小数的百分比 | `value.toFixed(2) + "%"` |

> 注：tooltip 中的公式描述值（如防御减伤率、抗性减免率）也遵循百分比取整规则，保留两位小数百分比。

#### 预览显示（xx+y 格式）中的取整

- `xx`（已确认值）按取整规则格式化后显示
- `y`（预览增量）也按取整规则格式化后显示
- 示例：真气上限已确认 `10`，本次增量 `2.5`，显示为 `10+2`（增量和已确认值均 floor 取整）

### 数据持久化

#### 存储方案
- **存储方式**：localStorage
- **存储时机**：每次点击"确认加点"后
- **存储内容**：当前已确认的元素加点数与所有一级属性的最终值
- **恢复机制**：页面加载时读取 localStorage，若存在则恢复为上次确认的状态
- **重置机制**：点击"重置"按钮后，清除 localStorage 数据，恢复为 `attributeInitConfig` 中的初始值

#### 存储数据结构
```json
{
  "version": 1,
  "elementPoints": {
    "fire": 0,
    "earth": 0,
    "metal": 0,
    "water": 0,
    "wood": 0
  },
  "attributes": {
    "attack": 0,
    "hp": 0,
    "defense": 0,
    "...": 0
  }
}
```

### 状态管理设计

数据流为三层结构：

```
┌──────────────────────────────────────────┐
│           confirmedState（已确认状态）        │
│   每次点"确认加点"后更新，持久化到 localStorage  │
│   所有一级属性的最终值、各元素已加点数           │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│         previewDelta（预览增量）              │
│   用户当前操作的未确认增量                      │
│   存储：每个基础元素本次临时加点的净点数          │
│   展示为 xx+y 中的 y                         │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│            displayState（显示值）             │
│   = confirmedState + previewDelta * coefficients │
│   界面实际展示的值                             │
│   计算属性（元素强度）在此基础上实时计算         │
└──────────────────────────────────────────┘
```
